## Baseline

The shared baseline is a 4-layer MLP (3072 → 1024 → 512 → 256 → 10) trained
on a 15,000-sample subset of CIFAR-10 for 50 epochs using SGD with a learning
rate of 0.01 and no regularization. A proper 3-way data split was used:
15,000 training, 5,000 validation, and 10,000 test samples. The validation set
was used exclusively for monitoring generalization during training, with the
test set evaluated only once after training completed.

The baseline reached **84.87% training accuracy** and **43.84% validation
accuracy**, producing a **generalization gap of 41.03%** — confirming strong
overfitting and providing a clear signal for regularization to act on.

---

## Implementation

### L1 Regularization

L1 adds a penalty proportional to the sum of absolute weight values to the
loss function at each training step:

$$\mathcal{L}_{total} = \mathcal{L}_{CE} + \lambda \sum |w_i|$$

This was implemented manually by computing the penalty and adding it to the
cross-entropy loss before calling `backward()`. The key property of L1 is that
it induces **sparsity** — it pushes many weights to exactly or near zero,
effectively removing redundant connections from the network.

```python
def l1_penalty(model):
    return sum(w.abs().sum() for w in model.get_linear_weights())

loss = loss_fn(outputs, labels) + L1_LAMBDA * l1_penalty(model)
```

### L2 Regularization

L2 adds a penalty proportional to the sum of squared weight values:

$$\mathcal{L}_{total} = \mathcal{L}_{CE} + \lambda \sum w_i^2$$

This was implemented using PyTorch's native `weight_decay` parameter in the
SGD optimizer, which applies L2 decay directly during the parameter update
step — the standard and numerically stable approach.

```python
optimizer = optim.SGD(model.parameters(), lr=0.01, weight_decay=L2_WEIGHT_DECAY)
```

Unlike L1, L2 does not produce sparsity — it shrinks all weights uniformly
toward zero without eliminating them entirely.

---

## Hyperparameter Tuning

Selecting the right penalty strength (λ) required experimentation. An
initial value of λ = 1e-4 for both techniques proved too weak — L2 in
particular showed almost no effect, with the gap remaining nearly identical
to baseline. Setting λ too high for L1 (1e-3) caused the penalty to
overwhelm the cross-entropy signal entirely, collapsing weight norms and
preventing learning.

The final values were selected after iterative testing:

| Technique | λ | Outcome |
|---|---|---|
| L1 | 1e-4 | Strong gap reduction, stable learning |
| L2 | 1e-3 | Mild gap reduction |
| L2 | 5e-3 | Stronger gap reduction, comparable to L1 |

## Results

All three models were initialized with identical weights using a fixed random
seed to ensure fair comparison.

| Model | Train Acc | Val Acc | Gap |
|---|---|---|---|
| Baseline | 84.87% | 43.84% | 41.03% |
| L1 (λ=1e-4) | 65.58% | 40.46% | 25.12% |
| L2 (λ=1e-3) | 80.43% | 43.36% | 37.07% |
| L2 (λ=5e-3) | 68.93% | 41.22% | 27.71% |

**L1** reduced the generalization gap by **39%** (41.03% → 25.12%). Training
accuracy was substantially constrained while validation accuracy remained
comparable to baseline, confirming that L1 prevented memorization without
hurting generalization ability.

**L2** with a weak lambda had minimal effect. With a stronger lambda (5e-3),
L2 achieved a gap of 27.71% — close to L1's result — demonstrating that L2
effectiveness with SGD is more sensitive to penalty strength than L1.
